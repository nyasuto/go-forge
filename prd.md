# PRD: GoForge — Go UNIXツールチェーン量産プロジェクト

## 概要

GoForgeは、UNIX標準コマンドおよび日常的に使うCLIツールをGoでゼロから再実装するプロジェクトである。学習・理解を目的とした「再発明」であり、既存ツールの完全互換を目指すのではなく、コア機能の再実装＋独自の改良を加える方針とする。

各ツールは独立したGoモジュール（シングルバイナリ）として実装し、Ralph Loopによる自律的な量産開発に最適化した設計とする。

## プロジェクト名

**GoForge** — Goで鍛造するツール群

## 設計原則

- **シングルバイナリ**: 各ツールは `go build` で単一バイナリを生成
- **ゼロ外部依存**: 標準ライブラリのみ（例外は明示的に記載）
- **テスト駆動**: 各機能にテーブルドリブンテストを必須とする
- **Progressive Enhancement**: Tier 1（コア機能）→ Tier 2（拡張）→ Tier 3（独自改良）の段階的実装
- **UNIX哲学準拠**: stdin/stdout パイプ対応、終了コード準拠

## ツール一覧と実装スコープ

### Wave 1: 基礎テキスト処理（Loopウォーミングアップ）

| ツール | 説明 | Tier 1（コア） | Tier 2（拡張） | Tier 3（独自改良） |
|--------|------|---------------|---------------|-------------------|
| **gf-cat** | ファイル結合・表示 | ファイル連結、stdin対応 | `-n` 行番号、`-s` 空行圧縮 | シンタックスハイライト自動検出 |
| **gf-head** | 先頭表示 | `-n` 行指定 | `-c` バイト指定、複数ファイル | ストリーミングモード（tail -fの逆） |
| **gf-tail** | 末尾表示 | `-n` 行指定 | `-f` フォロー | `-f` + パターンマッチでハイライト |
| **gf-wc** | 行・単語・バイト数カウント | 行数、単語数、バイト数 | 文字数（rune対応）、複数ファイル合計 | 日本語単語数推定、JSON出力 |
| **gf-tee** | 分岐出力 | stdout + ファイル書き出し | `-a` append | 複数ファイル、タイムスタンプ付与 |

### Wave 2: 検索・フィルタリング

| ツール | 説明 | Tier 1 | Tier 2 | Tier 3 |
|--------|------|--------|--------|--------|
| **gf-grep** | パターン検索 | 固定文字列マッチ、行出力 | 正規表現、`-i` `-v` `-c` `-n` `-r` | `-j` JSONフィールド指定検索 |
| **gf-find** | ファイル検索 | 名前マッチ、再帰探索 | `-type` `-size` `-mtime` | `-exec` の安全版、Glob対応 |
| **gf-sort** | ソート | 辞書順ソート | `-n` 数値、`-r` 逆順、`-k` キー指定 | 日本語読み仮名ソート |
| **gf-uniq** | 重複除去 | 隣接重複除去 | `-c` カウント、`-d` 重複のみ | 非隣接重複除去モード |
| **gf-cut** | フィールド切り出し | `-d` `-f` デリミタ＋フィールド | `-c` 文字位置 | CSV/TSV対応モード |

### Wave 3: テキスト変換・高度な処理

| ツール | 説明 | Tier 1 | Tier 2 | Tier 3 |
|--------|------|--------|--------|--------|
| **gf-sed** | ストリームエディタ | `s/pattern/replace/` 基本置換 | アドレス指定、`-i` in-place | マルチバイト安全な置換 |
| **gf-xargs** | 引数構築・実行 | stdin→引数変換、コマンド実行 | `-n` 分割数、`-P` 並列実行 | `-0` null区切り、dry-runモード |
| **gf-diff** | 差分表示 | 行単位diff (Myers algorithm) | unified format | カラー出力、単語単位diff |
| **gf-tree** | ディレクトリツリー表示 | 再帰的ツリー描画 | `-L` 深さ制限、`-I` 除外パターン | ファイルサイズ・Git status表示 |
| **gf-jq** | JSONプロセッサ | `.key` `.key.nested` `.[0]` 基本パス | パイプ `\|`、配列操作 `map` `select` | YAML入力対応 |
| **gf-hexdump** | バイナリダンプ | 16進ダンプ表示 | ASCII併記、オフセット | カラー出力、パターンハイライト |

## 各ツールの共通仕様

### CLI引数パース
- `flag` 標準パッケージのみ使用
- `--help` で usage 表示
- `--version` でバージョン表示
- 未知のフラグはエラー終了（exit code 2）

### I/O規約
- 引数なし or `-` → stdin から読み取り
- 引数あり → ファイルパスとして処理
- 出力は常に stdout（エラーは stderr）
- パイプ対応必須（`isatty` チェックでカラー出力の自動切替）

### エラーハンドリング
- ファイルが存在しない → stderr にメッセージ、exit code 1
- パーミッションエラー → stderr にメッセージ、exit code 1
- パイプ破断（SIGPIPE）→ 静かに終了

### テスト要件
- 各ツールに `main_test.go` を必須とする
- テーブルドリブンテスト形式
- 最低テストケース: 正常系3、異常系2、エッジケース2（空入力、巨大入力、マルチバイト）
- `go test ./...` で全テスト実行可能であること


## 成功基準

- 15ツール以上を実装完了
- 全ツールで `go test` がパス
- 各ツールがパイプチェーンで連携動作すること（例: `gf-find . | gf-grep "\.go$" | gf-wc -l`）
- 自分の日常作業で最低3つは実際に使えるクオリティ

## 技術的制約

- Go 1.22+
- 外部依存ゼロ（標準ライブラリのみ）
- `internal/` に共通コードを置く場合は最小限に留める
- 各ツールは独立して `go build` 可能であること
