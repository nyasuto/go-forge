# タスク

CLAUDE.md を読んでコンテキストを理解せよ。

## ルール
- 1回のループでは1つのタスクのみ実施
- 実装後、必ずテストを実行
- もしエラーにハマって解決策を見つけた場合や、このプロジェクト特有のベストプラクティスに気づいた場合、何かよく起きる問題に対する再発防止策を思いついた場合、終了する前に必ず CLAUDE.md に『サイン（次回の自分への注意書き）』として追記しなさい

あなたは自律型のソフトウェアエンジニアです。以下の手順に厳密に従って、タスクを1つだけ実行して終了してください。

1. `prd.md`（要件定義とタスク）と `progress.md`（現在の進捗）を読み込んでください。
2. `prd.md` のタスクリストから、まだ完了していないタスクを上から順に1つだけ選んでください。
3. 選んだタスクのコードを実装し、ローカルでテストや実行を行って動作確認してください。
4. 実装が完了したら、`progress.md` に完了したタスクと作業内容を追記してください。
5. `prd.md` のタスクリストのチェックボックスを `[x]` に変更して保存してください。
6. ここまでの変更を `git add .` および `git commit -m "feat: タスク完了"` でコミットしてください。
7. すべての作業が終わったら、それ以上の作業は行わず、直ちに完了を報告して終了してください。
