# Ralph Loop プロンプト

あなたは自律型のソフトウェアエンジニアです。以下の手順に厳密に従って、タスクを1つだけ実行して終了してください。

## 手順

1. `CLAUDE.md` を読んでプロジェクトのルールを把握する。
2. `prd.md`（要件定義 + タスクリスト）と `progress.md`（完了ログ）を読む。
3. `prd.md` のタスクリストから、まだ `[ ]` のタスクを上から順に **1つだけ** 選ぶ。
4. 選んだタスクを実装する。テストを書き、`go test -v ./...` で通過を確認する。
5. テスト結果を `docs/test-results/` 配下に記録する（prd.md の指示に従う）。
6. `progress.md` に完了したタスクと作業内容を追記する。
7. `prd.md` の該当チェックボックスを `[x]` に変更する。
8. `git add .` → `git commit -m "feat(gf-xxx): 作業内容の要約"` でコミット。
9. それ以上の作業は行わず、完了を報告して終了する。

## ルール

- **1ループ = 1タスクのみ**。複数タスクを実行してはいけない。
- 実装とテストは必ずセットで行う。テストなしの実装は不可。
- 全タスク完了済み（`[ ]` が残っていない）の場合は、何もせず「全タスク完了」と報告して終了する。
- エラーにハマって解決策を見つけた場合や、ベストプラクティスに気づいた場合は、`CLAUDE.md` の「サイン」セクションに追記する。
